AUnderstanding the Layout Pattern

Goal: Understand the desktop sidebar pattern and how it transforms for mobile devices.

üìã The Pattern:

A sidebar navigation is perfect for desktop screens where horizontal space is abundant. On mobile devices, however, a fixed sidebar would consume too much screen real estate. The solution? Transform it into a top navigation bar with a collapsible menu.

Main Content Area

This is where your page content lives. The sidebar stays fixed on the left, providing easy access to navigation.

ü§î Why transform the sidebar?

  • Space efficiency: Mobile screens are narrow‚Äîa sidebar would take up 40-50% of the width
  • User expectations: Mobile users expect top navigation or hamburger menus
  • Touch targets: Vertical menus work better with thumb-friendly tap zones
  • Content priority: On mobile, content should be the focus, not navigation

BBuild the HTML Structure

Goal: Create semantic HTML that supports both desktop sidebar and mobile navigation layouts.

üìã Your HTML should include:

  • A container <div class="layout-container"> to hold everything
  • An <aside> element with class sidebar for navigation
  • A checkbox and label for mobile menu toggle (the "checkbox hack")
  • A <main> element for page content
  • Multiple content sections to demonstrate scrolling

    </nav>

  <!-- Main Content -->
  <main class="main-content">

CStyle the Desktop Sidebar Layout

üìö Research these properties:

EAdd Media Query for Mobile Layout

Goal: Transform the sidebar into an off-canvas mobile menu at 768px and below.

üìö Research these concepts:

  • @media queries
  • max-width breakpoints
  • transform: translateX()
  • Off-canvas navigation patterns

üé® Your Task:

Add this media query to transform the layout on mobile:

@media (max-width: 768px) {
  /* Show mobile header */
  .mobile-header {
    display: flex;
  }

  /* Remove sidebar offset from main content */
  .main-content {
    margin-left: 0;
  }

  /* Transform sidebar into off-canvas menu */
  .sidebar {
    transform: translateX(-100%);
    transition: transform 0.3s ease;
    width: 280px;
  }

  /* Show close button in sidebar */
  .close-btn {
    display: block;
    position: absolute;
    right: 1.5rem;
    top: 2rem;
    font-size: 2rem;
    cursor: pointer;
    background: none;
    border: none;
    color: white;
    line-height: 1;
  }

  /* When checkbox is checked, slide sidebar in */
  .sidebar-toggle:checked ~ .sidebar {
    transform: translateX(0);
  }

  /* Add overlay when menu is open */
  .sidebar-toggle:checked ~ .sidebar::before {
    content: '';
    position: fixed;
    top: 0;
    left: 280px;
    right: 0;
    bottom: 0;
    background: rgba(0,0,0,0.5);
    z-index: -1;
  }
}

üéâ What just happened?

  1. Desktop (>768px): Sidebar visible, mobile header hidden
  2. Mobile (‚â§768px): Sidebar slides off-screen to the left
  3. Mobile header appears with hamburger icon
  4. Click hamburger: Checkbox toggles, sidebar slides in
  5. Dark overlay appears behind sidebar using ::before

‚ö†Ô∏è The order matters!

In your HTML, the checkbox must come BEFORE the sidebar in the DOM. The :checked selector uses the sibling combinator (~) which only works with elements that come after the checkbox.

FAnimate the Hamburger Icon

Goal: Transform the hamburger icon into an X when the menu is open.

üé® Your Task:

Add these styles inside your mobile media query:

/* Hamburger animation to X */
.sidebar-toggle:checked ~ .main-content .hamburger span:nth-child(1) {
  transform: rotate(45deg) translate(7px, 7px);
}

.sidebar-toggle:checked ~ .main-content .hamburger span:nth-child(2) {
  opacity: 0;
}

.sidebar-toggle:checked ~ .main-content .hamburger span:nth-child(3) {
  transform: rotate(-45deg) translate(7px, -7px);
}

üí° How the animation works:

  • Top line: Rotates 45¬∞ and moves down-right to form top of X
  • Middle line: Fades out completely
  • Bottom line: Rotates -45¬∞ and moves up-right to form bottom of X
  • The transition we added earlier makes it smooth!

GClose Menu When Link is Clicked

Goal: Automatically close the mobile menu when a user clicks a navigation link.

üíª The Problem:

Right now, when you click a nav link on mobile, the page scrolls to the section but the sidebar stays open. Users have to manually close it ‚Äî not ideal!

üé® Your Task:

Add this JavaScript before your closing </body> tag:

<script>
  // Close mobile sidebar when a navigation link is clicked
  const sidebarLinks = document.querySelectorAll('.sidebar-nav a');
  const sidebarToggle = document.getElementById('sidebar-toggle');

  sidebarLinks.forEach(link => {
    link.addEventListener('click', () => {
      // Uncheck the checkbox to close sidebar
      sidebarToggle.checked = false;
    });
  });
</script>

üí° How it works:

  1. Find all links in the sidebar navigation
  2. Add a click listener to each link
  3. When clicked, uncheck the checkbox
  4. CSS sees unchecked state and hides the sidebar

üé® Bonus: Highlight Active Section

Add this JavaScript to highlight the current section in the navigation:

<script>
  // Highlight active section in navigation
  const sections = document.querySelectorAll('section[id]');
  const navLinks = document.querySelectorAll('.sidebar-nav a');

  window.addEventListener('scroll', () => {
    let current = '';

    sections.forEach(section => {
      const sectionTop = section.offsetTop;
      const sectionHeight = section.clientHeight;
      if (window.pageYOffset >= sectionTop - 300) {
        current = section.getAttribute('id');
      }
    });

    navLinks.forEach(link => {
      link.classList.remove('active');
      if (link.getAttribute('href') === '#' + current) {
        link.classList.add('active');
      }
    });
  });
</script>

IAccessibility & Final Touches

Goal: Ensure your navigation is accessible to all users, including those using keyboards and screen readers.

üé® Add ARIA Labels:

Update your HTML with accessibility attributes:

<!-- Checkbox with ARIA -->
<input type="checkbox" id="sidebar-toggle"
       class="sidebar-toggle" aria-label="Toggle navigation menu">

<!-- Hamburger with ARIA -->
<label for="sidebar-toggle" class="hamburger"
       aria-label="Open navigation menu">
  <span></span>
  <span></span>
  <span></span>
</label>

<!-- Close button with ARIA -->
<label for="sidebar-toggle" class="close-btn"
       aria-label="Close navigation menu">×</label>

<!-- Sidebar with role -->
<aside class="sidebar" role="navigation" aria-label="Main navigation">
  <!-- ... -->
</aside>

üé® Add Focus Styles:

Make keyboard navigation visible:

/* Focus styles for keyboard navigation */
.sidebar-nav a:focus-visible {
  outline: 3px solid #667eea;
  outline-offset: -3px;
}

.hamburger:focus-visible {
  outline: 3px solid #667eea;
  outline-offset: 3px;
}

.close-btn:focus-visible {
  outline: 3px solid white;
  outline-offset: 3px;
}

/* Remove default focus styles (we've added better ones) */
*:focus:not(:focus-visible) {
  outline: none;
}

üé® Prevent Body Scroll When Menu is Open:

On mobile, prevent background scrolling when sidebar is open:

<script>
  const sidebarToggle = document.getElementById('sidebar-toggle');

  sidebarToggle.addEventListener('change', () => {
    if (sidebarToggle.checked) {
      // Menu is open - prevent body scroll
      document.body.style.overflow = 'hidden';
    } else {
      // Menu is closed - restore scrolling
      document.body.style.overflow = '';
    }
  });
</script>
