AUnderstanding the Layout Pattern

Goal: Understand the desktop sidebar pattern and how it transforms for mobile devices.

ğŸ“‹ The Pattern

A sidebar navigation is perfect for desktop screens where horizontal space is abundant. On mobile devices, however, a fixed sidebar would consume too much screen real estate. The solution? Transform it into a top navigation bar with a collapsible menu.

Why This Pattern Works:

  • Space efficiency: Mobile screens are narrowâ€”a sidebar would take up 40-50% of the width
  • User expectations: Mobile users expect top navigation or hamburger menus
  • Touch targets: Vertical menus work better with thumb-friendly tap zones
  • Content priority: On mobile, content should be the focus, not navigation

EAdd Media Query for Mobile

Goal: Transform the sidebar into an off-canvas mobile menu at 768px and below.

ğŸ¨ Mobile Media Query

@media (max-width: 768px) {
  /* Show mobile header */
  .mobile-header {
    display: flex;
  }

  /* Remove sidebar offset from main content */
  .main-content {
    margin-left: 0;
  }

  /* Transform sidebar into off-canvas menu */
  .sidebar {
    transform: translateX(-100%);
    transition: transform 0.3s ease;
    width: 280px;
  }

  /* Show close button in sidebar */
  .close-btn {
    display: block;
    position: absolute;
    right: 1.5rem;
    top: 2rem;
    font-size: 2rem;
    cursor: pointer;
  }

  /* When checkbox is checked, slide sidebar in */
  .sidebar-toggle:checked ~ .sidebar {
    transform: translateX(0);
  }

  /* Add overlay when menu is open */
  .sidebar-toggle:checked ~ .sidebar::before {
    content: '';
    position: fixed;
    top: 0;
    left: 280px;
    right: 0;
    bottom: 0;
    background: rgba(0,0,0,0.5);
    z-index: -1;
  }
}

The Magic Explained:

  1. Desktop (>768px): Sidebar visible, mobile header hidden
  2. Mobile (â‰¤768px): Sidebar slides off-screen to the left
  3. Mobile header appears with hamburger icon
  4. Click hamburger: Checkbox toggles, sidebar slides in
  5. Dark overlay appears behind sidebar using ::before

The checkbox must come BEFORE the sidebar in your HTML. The :checked selector uses the sibling combinator (~) which only works with elements that come after the checkbox.

GClose Menu on Link Click

Goal: Automatically close the mobile menu when a user clicks a navigation link.

ğŸ’» Add JavaScript

This requires JavaScript â€” pure CSS can't handle this interaction:

<script>
  // Close mobile sidebar when a navigation link is clicked
  const sidebarLinks = document.querySelectorAll('.sidebar-nav a');
  const sidebarToggle = document.getElementById('sidebar-toggle');

  sidebarLinks.forEach(link => {
    link.addEventListener('click', () => {
      // Uncheck the checkbox to close sidebar
      sidebarToggle.checked = false;
    });
  });
</script>

JavaScript Logic:

  1. Find all links in the sidebar navigation
  2. Add a click listener to each link
  3. When clicked, uncheck the checkbox
  4. CSS sees unchecked state and hides the sidebar
